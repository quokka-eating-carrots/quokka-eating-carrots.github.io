220927 HEXO

Posted on 2022-09-27 In IT , Front-end

안녕하세요, 당근 먹는 쿼카입니다!
git을 이용하여 HEXO 블로그 만드는 법에 대해서 설명해 보겠습니다.

해당 설명은 git, node.js npm이 설치되어 있다는 전제로 진행합니다.

1. git bash에 HEXO 설치하기

우선 git bash를 통해 본인이 진행하려고 하는 폴더에 들어갑니다.
폴더를 선택 후 $ npm install -g hexo-cli을 작성하여 hexo를 설치합니다.

설치가 완료되었다면 진행하려고 하는 폴더를 hexo 환경에 맞게 초기화를 하여야 합니다.

해당 명령을 수행한 후 git에 ls를 입력하면 다양한 폴더가 생성된 것이 보일 겁니다.

그 중 _config.yml은 환경설정 파일, source/_posts 폴더는 블로그에 작성한 포스트들이 저장되는 경로입니다.

2. 블로그 환경설정 변경하기

_config.yml로 진입하여 환경설정을 변경해 줍니다.
제가 사용한 방법은 hexo blog 폴더에서 vi _config.yml 명령을 사용했습니다.

title은 본인이 사용할 블로그 이름
subtitle은 본인이 사용할 블로그의 부제목
description은 본인이 사용할 블로그 설명
author은 본인의 이름 혹은 닉네임

초기 설정으로는 간단하게 네 가지 정도를 바꿀 수 있습니다.

더 아래로 내려가면
url 본인이 사용할 url
(저는 github repo를 사용하였기 때문에 quokka-eating-carrots.github.io를 사용하였습니다.)

theme은 본인이 사용할 테마 (hexo 홈페이지에 theme 탭에서 사용법을 따르면 됩니다.)

github 주소 넣는 법 추가

3. 블로그 글 쓰기

$ hexo new [layout] <title>
을 사용하면 새로운 포스트가 만들어집니다.
title 부분에 띄어쓰기를 사용하게 된다면 꼭 “”을 사용하여 제목을 감싸 주세요.

source/_posts 경로로 들어가 본인이 만든 포스트를 켭니다.
저의 경우 vi <title>으로 들어갔습니다.

title categories tag 등을 작성하고 --- 아래로 본인이 작성하고 싶은 블로그의 내용을 씁니다.

4. 작성된 글 확인하기

작성된 글을 확인하기 전에 서버를 설치해 줘야 합니다.
본인이 만든 블로그 경로에 들어가 $ npm install hexo-server --save 로 서버를 설치해 줍니다.

$ hexo server로 명령을 하여 서버를 열어 본인이 작성한 글이 잘 적용되었는지 확인해 봅니다.

http://localhost:4000가 기본값입니다. 서버가 열렸을 때 ctrl+c를 누르게 되면 서버가 꺼지기 ㄷ때문에 주의하세요.

5. 블로그에 글 올리기

$ hexo clean $ hexo generate 명령을 하면 글을 생성하게 됩니다.
그 후 $ hexo deploy를 사용하여 최종적으로 글을 생성합니다.

본인의 블로그 링크로 들어가 작성된 글을 확인합니다.

지금까지 hexo 블로그 사용에 대한 포스트였습니다.
감사합니다!
